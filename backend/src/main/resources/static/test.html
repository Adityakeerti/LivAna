GET Rate Limit Stress Test — /api/users/me (no auth)

Fires sequential unauthenticated requests to /api/users/me. This endpoint is in our Security chain — the UNAUTH tier (30 req/min per IP) applies. The 31st request should return 429 Too Many Requests with a Retry-After header.

Why /api/users/me? /actuator/health is handled by Spring Boot's management security chain (separate from our custom chain), so our RateLimitFilter may not intercept it. /api/users/me definitely goes through our filter chain.

Check server logs for: [RateLimit] GET /api/users/me → tier=UNAUTH key=rl:ip:...
After test, run in Valkey: redis-cli --scan --pattern 'rl:*'
